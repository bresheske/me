angular routing with github pages

August 23rd, 2017


So by default, github doesn't allow any additional routing or URL rewriting. It will allow you to enter into new routes and the URL will follow, but it won't know what to do if you attempt to GET a sub route (for example, you refresh the page). Github will return a 404.

but! github allows custom 404 pages!

A very quick and dirty solution is to simply copy and paste your index.html file into 404.html. This way your application will load up normally and will take the subroute into account.

check out this website

  • Press F12 and open up your console.
  • Press F5 to refresh your browser.

Notice you'll see github actually returns a 404 error when a subroute is requested, but the 404 page served is our main application. It's a bit hacky, but all is well with the world.

one caveat

This only works if your URL resolves to a 404 error. So, if files or folders match your route names, github will serve up those first.
